## 文章结构

### 1. 引子（~200 字）
简述 hookrace.net 的真实案例：DDNet 游戏服务器被人用伪造 IP 的端口扫描攻击，导致被主机商关停。引出问题——有人能用你的 IP 干坏事，而你毫不知情。

### 2. IP 欺骗扫描原理（~400 字）
- 什么是 IP Spoofing，为什么 TCP 三次握手让伪造变得困难
- nmap 的两种伪装扫描：
  - Idle Scan（-sI）：利用僵尸主机的 IP ID 递增来间接判断端口状态，完全隐藏攻击者 IP
  - Decoy Scan（-D）：在真实扫描中混入大量伪造源 IP 的探测包，让目标无法分辨真正的扫描者

### 3. 实操演示：Decoy Scan（~500 字）
- 在本地环境中用 nmap -D 演示伪装扫描
- 用 tcpdump 同时抓包，对比显示目标服务器看到的"扫描来源"
- 命令示例 + 实际输出

### 4. 实操演示：Idle Scan（~400 字）
- 找一台适合的僵尸主机（如网络打印机等 IP ID 递增的设备）
- 演示 nmap -sI 的完整流程
- 用 tcpdump 展示数据包流向，说明为什么攻击者 IP 完全不出现

### 5. 防御方案（~500 字）
- 检测：用 tcpdump / netstat 分析异常流量模式
- iptables 规则：限制出站 TCP 新连接，只开放必要端口
- BCP38 / uRPF：解释为什么 ISP 层面的源地址验证才是根本解决方案
- 与主机商沟通：建议保留日志、主动说明情况

### 6. 小结（~150 字）
总结：IP 欺骗扫描利用的是信任链漏洞，个人能做的防御有限，根本解决需要网络运营商配合。
